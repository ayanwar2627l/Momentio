Backend Insight: What is authentication doing?
When a user registers:-

Frontend sends name, email, password
Backend checks if user exists
Backend hashes password
Backend saves user in MongoDB
Backend creates JWT token
Backend sends user + token to frontend

When a user logs in:
Frontend sends email + password
Backend finds user by email
Backend compares password
Backend creates JWT token
Backend sends token back


For protected routes:
Frontend sends token in headers
Backend verifies token
Backend allows/denies request

**Backend Insight: Why bcrypt?
Never store plain passwords.

normal password: "123456"

hashed password: "\$2a$10$sd8f..."
bcrypt.hash() converts the password into a secure hash.

Then during login:
bcrypt.compare(password, user.password)
checks if the entered password matches the hash.

**Backend Insight: What is JWT?
JWT is like a login pass.

After login, backend sends:
token
Frontend stores it and sends it with protected requests:
Authorization: Bearer token_here
Backend verifies the token to know who the user is.

**Backend Insight: What is middleware?
Middleware runs before your controller.

For example:
router.get("/me", protect, getMe);
This means:
First run protect
If token is valid, run getMe
If token is invalid, stop request
